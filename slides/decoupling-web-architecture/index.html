浅谈「前后端分离」

欧雷

Long long ago...

网页还不那么复杂

人们还不那么关注页面的颜值有多高

也对页面交互的舒适度要求不高

更不知 UX 为何物

只要能用就行……

如果是刚起步不久的小公司

整个网页的开发很可能让后端工程师全包了

于是页面就……

全栈式开发

当业务不断增多

当代码变得复杂

这时就需要专职的前端工程师出马了

也需要对整体架构做些调整

如果不想整个项目变得像一坨屎……

我觉得最好的方式是

「前后端分离」

前后端分离的基本方法是

将前端代码从后端框架中抽取出来

即，

把「V」和「C」都交给前端工程师处理

这个行为体现了

「关注点分离」

使 web 开发人员更关注自己的职责所在

目前有两种前后端分离的方式

单页面应用

如：

Backbone、Angular、React、Vue

Node.js 做中间层

如：

淘宝的「中途岛」项目

我们为什么要分离？

快速应变

  • 很多活动页或宣传页
  • 经常有文本变动或其他小修改
  • 页面发布流程需要独立出来

分清职责

  • 专（业）人做专（业）事
  • 渲染方案任意选
  • 以 API 文档为中心进行交流

弹性架构

  • 积累前端公共库
  • 终端媒介零关心
  • 随意插拔和拼装

如何去分离？

部署发布

  • 与 Java 应用通信
  • 配置 Nginx 规则

数据接口

  • 制定契约
  • 生成文档
  • 开发调试
