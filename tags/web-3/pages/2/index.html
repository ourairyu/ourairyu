借由 OpenBuild 训练营的实战作业，用了差不多 5 天时间从完全的智能合约小白到完成了自己第一个 dApp——

第一天学 Solidity 并写一部分智能合约。

第二天写完智能合约，了解 Hardhat 后编写 Ignition 模块。

第三天写智能合约测试，利用 Ant Design Web3 完成前端基本样式与交互逻辑。

第四天完善智能合约测试，本地部署智能合约后前后端联调部分功能。

第五天完善功能并本地调试通过后将智能合约部署到 Sepolia 以太坊测试网。

至此，我的第一个 NFT 市场应用，同时也是我的第一个 dApp，完成！Web 3Ant Design

打算入去中心化应用的坑，于是今天作为用户体验了下 Web3 生态——

每个 dApp 都需要钱包作为身份校验工具，而钱包真的是五花八门、玲琅满目，令人难以抉择；还好有 WalletConnect 这个通用协议，在选择钱包上就没那么纠结了，最终选了 OneKey。

对于用户侧的实际体验就是区块链地址替代了账号密码（暂时没用金融功能），其他与传统的没啥差异。

总记账号密码是挺烦的，为啥钱包不能跟 Authenticator、密码管理工具进行统合，从而打通中心化与去中心化应用的使用体验？

Mask Network 声称是 Web 2 到 Web3 的 portal，然而体验后并不符合我的预期……去中心化Web 3区块链
