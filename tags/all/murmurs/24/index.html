有种说法，人脑对身体起到「限制器」的作用，也就是说我们平时所表现出的身体能力是受限的，仅在特殊时刻或通过特殊手段才能激发完全或接近完全的能力，像遇险时、愤怒时等。

如果长时间、频繁地突破限制，会给身体造成很大负担，甚至会减寿。《火影忍者》中凯老师和小李所使用的「八门遁甲」可以说很好地诠释了这点，每个「门」就是一个重要的「限制器」，当八个「门」全部解开，就是放弃自己的生命了。大脑火影忍者

设计一个按钮组件很简单吗？反正我是觉得挺难的——该有哪些特性？其语义是什么？哪些是合并成一个 prop，哪些是单独成为 prop？

光一个 type prop 我就纠结半天——

这个 prop 的语义到底是什么？是风格类型？还是功能类型？或是其他？

最流行的国产两个组件库中，Element 按钮组件 type 的值是 primary / success / warning / danger / info / text，姑且可以认为是「状态类型」或「（Bootstrap）功能色类型」；而 Ant Design v4 中按钮组件 type 的值是 primary / ghost / dashed / link / text / default，我完全看不出来这是按照什么来进行分类的「类型」。虽然我认为在这点上 Ant Design 和 Element 都是失败的设计，但 Ant Design 尤为失败！Ant Design

从常见的技能树想到将知识点可视化，形成一个图形化的 3D 世界，点击某个点都可以下钻显示其包含的节点。

例如——

起初是一个地球🌍，如果点击海洋区域的某个点，会显示这个点是哪个大洋、属于哪个国家、海水组成成分、有哪些海洋生物等信息；点击陆地同理。

当选中人时，会围绕着人从历史、文化、生物、哲学、医学等方面显示相关信息。

说白了就是将人类的各种知识建立模型并进行关联，再通过 3D 建模的方式图形化表现出来。数据源可以是像维基百科这种，但远远不够。

这个想法💡的出现，让我知道了「知识图谱」、「知识可视化」等术语，并发现了「知识工程」这个新世界！知识工程知识图谱
