由于前天把 Handie 相关的所有源码和 demo 都集中在这一个 monorepo 中了，并且 demo 是分别基于 Vue 和 React 的用于展示和调试源码的前端 Web 应用，理论上逻辑、配置等需要一致，甚至是源于同一份文件，这就产生了两个主要的需求：

  1. 基于不同的运行时视图库/框架搭建的 demo 间应当共享同一份逻辑、配置等；
  2. mock 请求相关操作要能在 Node.js 内存、前端 JS 内存甚至是 IndexedDB 中进行。

需求 1 暂时是将共用部分提到 demo 应用文件夹之外后再在 demo 中引用，但觉得应该有个什么方式创建一个 headless app。

需求 2 则是先把原本在 Node.js 环境中的纯内存的简易 CRUD 代码转移出去作为通用 JS 环境的操作，这样一来，就算编译后的纯前端也能在浏览器内存中进行简易 CRUD 模拟。

基于此，也许本来不知何时才会去做的 ORM 可以提上日程了，一切想要进行数据持久化的操作 Handie 前端视图层都通过 ORM 去交互，至于到底最终是在 Node.js 或前端 JS 内存、IndexedDB 或是后端数据服务什么的，都可以根据开发阶段和编译环境等在编译期去决定！

如此一来，过后在做纯前端/无后端的在线编辑配置与 CRUD 操作的示例站点（playground）时就可以更简单地使用 IndexedDB 啦！并且，把这个功能应用到日常开发的可视化配置或搭建时，也能更方便地迁移到利用 dev server 存储为本地文件！前端开发中后台前端应用Handie
