如果把 HTML、CSS 和 JS 当成单纯的网页语言来看待，那么就只能用它们写写 web 页面；但若把 HTML 看作是描述视图结构的 DSL，把 CSS 看作是描述视觉效果的 DSL，把 JS 看作是可注入宿主环境且同时支持 OO 和 FP 的轻量级 GPL 的话，那么可玩性就无穷大了——

基于 HTML、CSS、JS 及它们的变体，可以满足三种场景的需求：

  1. 只有界面没有交互，像电子的图书、杂志、海报等以展示内容为主的，只需 HTML、CSS 或它们的变体；
  2. 只有交互没有界面，像 CLI 工具、服务端应用、IoT/WoT 应用等，只需注入了宿主环境的 JS，这类还被叫做「无头（headless）软件」；
  3. 既有界面又有交互，常规的客户端（无论是基于 web 还是原生）应用都是这种，会同时用到 HTML、CSS 或它们的变体以及（注入了宿主环境的）JS。

其中，1、3 需要排版引擎，2、3 需要 JS 引擎，所以 3 需要同时具备排版引擎和 JS 引擎。

在已知的满足第 3 种场景需求的应用类型中，浏览器就不用说了，那么在国内备受关注的小程序与浏览器和一般的混合应用分别有什么区别？

我认为，小程序本质上是在架构和工程方面进行了改进的混合应用，它实际上就是运行在 web view 里的页面；改进后的架构更复杂，整体上与浏览器有所相似。
