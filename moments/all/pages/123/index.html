事不宜迟🙂Fxxk Design

评论
2021-10-28 00:37 欧雷说

虽然一直在用，但最近突然就「顿悟」了——

在编程中，interface-based programming 与 DI 是很重要的思想，是其他一些针对具体场景的模式、方法的基本。

在这里，「interface-based programming」中的「interface」不是指 class-based OOP 中的 interface，而是相对抽象的像「约定」、「协议」、「规范」之类的概念，它确定了有交流的双方拿什么去交流；而「DI」则是决定了「interface」的具体指向——可以说它们是互为表里、形影不离的。

至于实例，最容易理解的就是 class-based OOP 中的 interface 与 class；像模板模式、观察者模式、DI 技术、DSL 等都是。

「interface」罗列了一堆「名词」，而「DI」定义了这些「名词」。

抓住 interface-based programming 与 DI 这个组合的核心思想，可以解决很多程序和架构问题。程序设计面向对象程序设计基于接口编程依赖注入
