一个 Git 项目一般来说我会至少推到两个托管平台上，如 GitLab 和 GitHub，有时也会多出来 Gitee 或其他的，因而代码同步是个刚需。

之前我都是手动同步，今天把流程优化了下，尽量引进自动化手段：

  1. 在 GitLab 上配置 GitHub 为镜像；
  2. 利用 GitHub Actions 把代码同步到 Gitee。

这样一来，我在本地只需配置 GitLab 为远程源即可。GitHub

昨日白天大部分时间在了解并试用扣子，探索发掘它的潜力，因而稍微做个总结吧——

目前扣子创建的 bot 可免费自定义的部分主要有：

  • Prompts
  • 插件列表
  • 工作流
  • 知识库
  • 数据库（存储用户输入文本解析出的结构化数据）

单从这些看，只要云雀大模型给力，可玩性还是比较可以的。

其发布渠道有：

  • 豆包
  • 微信服务号/微信客服
  • 飞书
  • 掘金

除了豆包之外，其他的对个人开发者可以说没卵用，且豆包现在好像也不算是主流的，所以扣子在应用层面暂时有点鸡肋。

在熟悉过程中，对于 bot 用来干嘛，基于我自己的情况，产生了一些点子或者说设想。

主要是想创造一个自己的数字分身，让别人跟它对话就相当于跟我对话。

鉴于它只能运用我已知的知识而无法产生新知识新想法，因而只能作为：

  • 简单咨询/提问的对象
  • 了解我的一个途径

无法作为讨论的对象，想讨论还是得找我本尊。

我觉得做个专属数字分身是内容创作者未来的一个新方向，是打造个人品牌的一个利器，是适应 AIGC 时代的必然。
