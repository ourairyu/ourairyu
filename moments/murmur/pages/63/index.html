想法相关的时刻

2020-08-07 16:42 欧雷说

之前觉得人在年轻时，最好是刚工作那会甚至更早，就应该计算下自己结婚生子、子女教育以及养老大概会用到多少钱。按照是否必要排序，养老最优先，结婚次之，子女最后。然后以这个金额为底线去理财投资。

这两天又觉得，不能觉得自己年轻、身体好就去挥霍，再年轻身体再好，也要坚持良好的作息和饮食。「身体是革命的本钱」之类的话听得耳朵都起了茧，但意识就是上不去……你不爱护身体，疾病早晚会找上你，没准就是在你还年轻时。

做好健康管理吧！除了保持良好的作息和饮食之外，保持良好的心态，少生气；尽可能锻炼，主要是心肺等，肌肉次之；买保险，做好防备，像重疾保险；每日早晚做血压、体温等检测并记录；每年做较为全面的体检。

人在年轻时最主要最重要的事，就是为了让自己能够在老年时尽可能过得舒服、快乐而做准备。

人工智能这种东西，从计算设备诞生的那一刻起，就是必然会出现的。

运转在计算设备上的是人造的虚拟世界，人就是这个「世界」的主宰，人的意识和行为就是这个「世界」的「道」。人必定会将自身投射到这个「世界」中，因为「人」是人所能想到的最高级的存在。

人高级就高级在能够对事物进行认知，也可以说是智能，其在这个「世界」中的映射就是人工智能，这也是为什么说现在的「AI」并不是真正的人工智能。

人要认知事物，首先要感知事物，感知的目的是收集数据，这是进一步进行整合成信息、沉淀为知识并内化为智慧的前提。

通常人是通过眼、耳、鼻、舌、身这五识来进行感知。对于人工智能来说，传感器、拾音器、摄像头这类自动收集数据的设备配以语音识别、图像识别等技术就相当于人的五识——这些再加上人工智能，基本可以模拟人的神经系统了——人工神经网络。

如果人造血管、人造器官等技术得到了十足的发展，再配上人工神经网络，就出现了「人造人」；若将人造血管、人造器官等用于活人，那就成了「改造人」。

2020-06-27 13:13 欧雷说

昨天开车跟🐷聊天时说：「我为自己已经跳出了框框感到高兴！」

这里所说的「框框」，是思维的局限。归功于这一两个月的思考和知识补充，虽然我的思维可能还是受限，但与之前相比毋庸置疑已经提高到了一个新境界。即使可能领悟得比很多同龄人晚了，甚至比很多年下者晚，但我觉得只要领悟了，就不算晚。

很多人都听过什么「道生一，一生二，二生三，三生万物」和「太极生两仪，两仪生四象，四象生八卦」之类关于从无到有再到无限的思想概要。这都是在说世界本源与发展规律的，是形而上的东西。

通俗点说，「从一到无限」就是从一个最原始的根基衍生出无数个具备相同或相似特性与逻辑又不完全相同的个体/实体。最能让所有人都能理解的，就是生物的繁衍生息与进化。

很多道理，是人们都听过都知道的，但真的意识到且有所体感，那是另外一回事。就像很常见的交通安全问题，信号灯和人车分流的规则傻子都知道，为什么还是有那么多行人、非机动车闯红灯或不在自己应该行走/行驶的路上？那是因为交通事故没有发生在他们身上，他们没有切身意识，没有体感。

说了这些，又跟「框框」，也就是说思维的局限有什么关系呢？就拿做 web 开发的人（前、后端都一样）来说——

最低级的开发者只考虑自己手头的事情，连自己负责的那块内容的后期维护成本都不考虑，代码的写法一点也不注意，为了实现需求使用很恶心的反模式。

高级些的开发者懂得封装、复用，会考虑协作效率和维护成本等问题，但其思维也局限于某一层，如视图层。

更高级的开发者具备架构意识，懂得应用分层并进行把控，抽象能力、管理能力更好，但其思维可能局限于前端/后端领域。

真正能把代码写好的，能做出好软件的人，他的思维必定不会局限于某个语言、某个框架、某个技术、某个端、某个领域，思维的高度必须是尽可能在「一」上，即「万物的本源」，而不是「无限」上，即「万物」。眼界越是在越末端，局限就越大，能做的事就越少，可能性就越小。

你在 web 前端遇到的问题，也许在原生客户端早就解决了；你在泛前端或 web 开发遇到的问题，也许在别的语言领域已经有方案了；你在软件开发遇到的问题，也许在建筑领域已经被解决——你的需求很大程度上不是你一个人的需求，你要解决的问题很大程度上不只是你这个领域的问题。

所以，我要说的是什么？我要说的就是：编程和软件开发难吗？不难！我们的主要关注点是编程和软件开发吗？不是！

想了下，人的认知应该是分为两类：概念性的，对一个事物是什么及其运作方式的理解，这是已知的，毋庸置疑的，即知识理论，作用是帮助人们沟通交流；理念性的，对一个事物该是什么及该如何运作的看法，这是未知的，会产生分歧的，即思维方式，作用是决定了一个人的决策倾向。

一个东西，如果只是去应用，那它就是工具，只需使用手册或技巧就行；如果是去研究，那它就是学问，需要相关的理论体系去支撑。

就像发的朋友圈，如果只是一扫而过，那它们就是文字、图片、视频；如果去琢磨一下，会发现里面有很多信息量。

词语的存在是为了方便人们交流，每个「词语」就是一个「数据包」，根据它所压缩的信息量，要真正理解一个词语可能需要大量的知识储备。

词语还蕴含了国家、民族文化，这也就是为什么有的外语词汇不好翻译。本国家、民族没有那种文化，很可能没有相应的词语，这时就得造词，并做一堆解释。

比喻就是一种关联模型的方式，像我前几天文章里说的「遇到新事物时可以关联现有模型去快速地理解并掌握它」，有些记忆法也是这种关联模型的方式。
