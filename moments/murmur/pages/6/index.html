  1. 基于 Web 标准，使用 Web Components 等而不用 React 之类；
  2. 结合 Web3 的 Web 3.0，打造 IndieWeb；

今天初次了解（还算不上学习）泰语，单从文字构成来看，比韩语复杂得多——共 44/42 个辅音/子音，和 32 个元音/母音，以及 5 个声调。

其中，辅音和元音的相对位置并非固定的，有一定的规则（具体暂未了解），并且有的发音相同或很接近，更增加复杂度。

但是，它不像英语和日韩语那样会根据时态等进行变形，而是像汉语一般由不同相对独立的语义单位任意组合，这样更为「亲切」。

泰语中的很多词汇是外来的，如梵语、巴利语、汉语等。

由于泰语的发音特点，想听起来很爷们儿是几乎不可能的……😂😂😂外语

昨晚发现发在掘金上的两篇分享自己向 Web3 转型经历的文章被「下架」处理，也就是除了我其他人完全看不到文章，并且系统通知中没解释原因，我就截图发到一个掘金群里要求行政层级较高的运营人员做出解释。

由于昨天是周日，工作人员不在线也可以理解，我就自己设置了一个今日内不给出合理解释就删号永久退出掘金的 deadline——当然，这并没有发到那个群里——工作日再不给出解释就说不过去了吧？

我不是什么网红、大咖、KOL 之类，人微言轻，设置 deadline 也不会对谁造成什么压力，只是表明下捍卫自尊的坚决立场与态度。

今天上午有官方运营人员在群中回复我了，但不是我艾特的人，他给的理由漏洞百出，根本无法让我信服，于是我们之间开始了一场「论战」。

期间也有个别其他看热闹不嫌事儿大的进来搅浑水，用同样像破渔网般的逻辑来给我隔靴搔痒，都懒得跟他们耍嘴皮子。

我的诉求很简单，以下两点二选一：

  1. 我违规了处理我可以，但让我明确无异议地知道文章的哪部分违了哪条规；
  2. 让我的文章可以正常被他人看到并访问，可以取消推荐。

反正，虽然群里那个运营人员没给出令我接受的解释，但看到我那两篇文章被放出来了，那我与掘金间的维权也算是告一段落，我可以不用删号而继续留在掘金上发内容了。

我没把平台方当成敌人，当然是希望它越来越好，不然为啥当初要做出在上面发内容的决定？但如果任意践踏我的尊严，不尊重我，同时还无视我的合理诉求，那就成了我的敌人！

不仅是我，对任何其他创作者同理。

鉴于我的诉求最终得到满足了，就不放出群聊记录供大家吃瓜鉴赏了～

这就是在中心化内容平台上创作的待遇——无理由处理。最晚明天不能给我个合理解释，只能删号退出了。[图片]查看详情
