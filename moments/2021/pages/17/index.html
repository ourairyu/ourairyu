无论是通过声音、文字还是肢体，「表达」都是既完整又不完整的——

说它是完整的，是说在表达一件事或一个概念时，要把这个要表达的中心相关的事物（如时间、地点、人物等）和逻辑（如关系、影响、前因后果等）都完整地阐述一遍，这样一来，虽然听的人不一定完全明白说的是啥，但会明白你要说啥。

听的人不一定完全明白说的是啥是因为表达不是完全完整的，只能相对的完整，就像一个系统中的子系统一样。不是完全完整就是说，在一次表达中会用各种名词去封装知识、概念，如果听的人不知道那些词的含义，就不会完全明白要表达的完整意思，也就是双方的背景知识或者说交流的上下文没有对齐。

「表达」是思维外化的体现，所以说，要提高表达能力就得先提升思维能力。

「反混沌」（Anti-chaos，a.k.a. NTKS）体系在工具方面目前主要有四条线——

「Petals」提供原子 UI 组件，可一定程度上定制，每个 UI 组件是一个包，核心与 React、Vue 之类的无关，任何人都可以基于核心选择自己渲染结构的方式。

「Buds」提供 UI 组件集成工具与服务，与「Petals」的理念结合，让「组件库」只限于个人或组织自己。

「Organik」是一个视图库无关的应用框架，像有机体一样。

「Handie」则是融合了上面每个的让各个技术栈（主要指视图库）在开发中后台应用时拥有几乎一致体验的开发框架。反混沌PetalsBudsOrganikHandie
