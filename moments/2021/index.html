知识图谱 + 流程图作为团队协作的 Single Source of Truth，既有利于业务知识沉淀，又能让产品们想清楚需求合不合理，还能提高不同环节人之间的协作效率。

这实际上就是领域建模的过程，知识图谱和流程图的维护，就是领域模型的维护，会与应用开发的低代码工具打通。

突然觉得 Web Components + Svelte 不错，这么看来我没啥可纠结的了——

单纯的 Web Components 没有响应式更新，如果要加的话，就是又做了一个 Svelte 类的东西，所以不如直接用。

对于使用者来说 Vue 和 Svelte 的体验是类似的，但 Svelte 更接近原生 Web 前端。

基于 Web Components 的直接就叫 handie，把 Svelte 作为增强附加项，而不是看做是一个单独的技术栈——这才是 Handie 的终极方案，对 Vue 和 React 组件的支持采取其他方式适配。HandieWeb ComponentsSvelte
