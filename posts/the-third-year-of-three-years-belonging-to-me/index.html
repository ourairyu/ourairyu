属于我的三年·第三年

欧雷 发表于 2022-12-23 13:46:59

0 条评论

标签：工作总结

作为「属于我的三年」中最后一年的总结，应该也是最后一次以这种「俗套」的方式写「年总结」——若是将总结平摊到日常，就没有在年底再来一次的必要。

我平时产生的数据的类别及数量已然很丰富，故打算今后换一种稍微新鲜点的方式——基于已有数据生成「个人年度报告」。

今年收获

不知为何，今年突然就越发觉得自己与周遭世界格格不入，一整年都过得挺难受的，时不时让我感到窒息。

并非是什么世俗层面的压力，可能是我的精神世界与周遭世界无法进行很好地调和而导致的。那种感觉真的很让人抓狂……😫😫😫

也许就是这个原因，让我在上半年基本一直处于心态上「躺平」的状态——

上班时不失职业素养地做好本职工作，适当体现出专业性、成长性，尽量保持平和；到下班点就立刻拎包走人，无谓加班是几乎不会去做的了。

不上班时大众所认为积极向上的事情我什么也不想做，像刻苦学习专业知识，好好写代码努力奋斗之类；反而热衷于那些看起来堕落、颓废而不求上进的事情，如打游戏，刷视频，追剧等等。

如今回头一看，去年总结「来年期望」中所说的一个也没去做——

让 Handie 在 Vue 和 React 应用中达到 production-ready 的状态，并提供可用的基于 Web Components 的多技术栈运行时共存方案。

将今年落下的学习研究「领域（domain）」相关知识、方法论、技术的计划补上，夯实自己在这方面的知识体系并提高认知。如果有可能，再去学习研究下类型系统相关知识、方法论、技术。

在前端团队和部门中通过培训、分享的方式帮助其他人理解领域驱动、模型驱动等思想，促进领域驱动团队的推动，为「前后端一体化」、「产研一体化」的落地做准备。

欧雷《属于我的三年·第二年》

虽说如此，这一年我也不是一无所获——

自我进化

由于上半年心态上的「躺平」使我慢了下来，不像以往那样跟打了鸡血似的在非工作时间疯狂地去开发自己的软件项目，而是留给自己更多放空的时间。

在这段时间里，感觉自己超越单纯观察力的感受能力变强了——就像自己身上那些原本封闭的门打开了，各种各样的事物进入其中，发生奇妙的反应后我的思绪就通了，也就悟了。

今年我大多时间都在放空，观察并感受外界事物与自己内心后进行思考，进而无论是在职业上还是在人生上，都在重新审视并反思后完成了自我进化——

从「软件工程师」变为（自创的）「反混沌工程师」——除了反软件开发中的混沌，还反人生中的混沌，其关键点就是「知识」。

这令我在更加看清周遭世界的同时也变得更为痛苦，因我暂时还未找到能够与之和善相处的方式。

相关思考可见「反思软件开发」系列文章与「Life-hacking」相关文章。

尝试转型

上半年的「躺」让我明白并想清楚了一些事情，于是下半年真正决定未来要作为创作者「卷」起来——

我的创作聚焦于「欧雷流人生工程」和「反混沌前端工程」这两个题材，内容的形式及范围为：

  • 图文——以体系化的思想理论为主，时机成熟可整理并发布/出版为电子书、纸质书；
  • 音频——针对某个话题在播客中进行杂谈或访谈；
  • 视频——日常、出行、旅游等生活记录，以及技术分享、教程讲解等。

这几种形式对「欧雷流人生工程」很是重要，于「反混沌前端工程」而言，提供企业级 Web 开发解决方案则是重点：

  • Fxxk Design——以「设计即代码」为目标的跨 JS-based 技术栈、跨平台 GUI 开发体系；
  • Future.js——以「需求即代码」为目标，将前端应用各个层次、各个环节之间的通信规范化、标准化的开发体系。
欧雷《不躺了，我要认真做创作！》

不能光耍嘴皮子，说做就做！

时隔多年，又重新拍起了 vlog，发布了几个相关视频；笔耕不辍，今年共发了近 30 篇文章，创历史新高！除此之外，还写了很多未公开发表的日常思考与笔记等。

没想到的是，《向过去的自己告别》这篇文章在微信公众号上竟成了我从来没有过的「爆款」——

现在还没在创作上放很多精力，当投入更多精力时，会让俺家🐷也参与进来，帮我分担一些后期、运营等方面的事情。

在工作方面，尝试向产品方向转型，也就是去思考一些与部门业务和能力相关的产品上的一些事情，并在合适的场合提出自己的观点与建议。

一开始是关于如何打造一个可以让更多人去用的知识库产品，在跟多人交流讨论，并得知明年知识组的「规划」之后，让我觉得这条路目前看来真的走不通。

后来觉得将「知识」与「智能」结合的一体化对外展示的「数据智能工作台」也许还有一线生机——让「智能」的「知识」指导大家日常工作与创新，可以促进部门内甚至公司内的数字化转型！

软件产品

简单搭建了跨 JS-based 技术栈控件（UI 组件）体系 Petals 与渐进式配置驱动企业级中后台前端应用开发框架 Handie 的官网，当前主要是起到文档查阅的作用。

将 Petals、Handie 及个人知识库相关的源码和数据分别进行整合形成 monorepo，提高内聚性并方便调试等操作。

对 Handie 进行了轻微地重构，变得更加灵活，能够覆盖更多的场景，减少了很多自定义部件的编写工作，节省了更多的代码，让模块可复用性和代码可维护性进一步提高！

基于 Handie 封装了一个部门级的业务应用 starter，将业务应用间相通、重复的二方包、三方包、代码逻辑、主题配置等统一管理起来，减少业务应用中的模板代码。

此时，这个 starter 会起到应用基座的作用，那些业务应用就基本是在这个通用基座上搭出来的纯业务向和业务应用独有的一些配置和文件。

这样一来，能够很好地抽象并隔离部门内业务应用间的共性及各应用独有的个性——该 starter 实际上就是本地化的低代码平台。

经过这一两年的努力，部门内还存活的业务应用的前端部分很大程度上是在 Handie 的能力的基础上配置出的，无论是 Vue 的还是 React 的。

在个人知识库体系方面，我创立了开放的个人知识库产品「KnoSys」——不仅读音与「gnosis」一致，可以按「知识」、「灵知」理解；还能看成是「knowledge」与「system」的结合，取「知识体系」或「知识系统」之义。

当前它还仅仅是占个坑的状态，尚未进行塑造。

来年期望

上文说了，今年一整年都让我十分难受，因此从明年开始的主旋律就是「破局」——

很多人处于困局之中，但大多数没有去找或找不到破局的方法。所谓「破局」，具有一定的颠覆性，并要承担一些风险，单纯的「解决方案」我不认为是「破局」。破局需要从局外找方案，解决相对根本的问题。

「破局」不是单点优化，而是结构性变革。

想活成自己，很多事情需要自己去找答案，而非按照已有的「标准答案」去做事。

欧雷的想法

也就是说，我要跳脱出当前的「主流」，另辟蹊径寻找符合并适合自己的叫做「欧雷流」的道路。

「主流」之路虽然相对安稳，但充满桎梏；「欧雷流」之路虽说自由，但会沐浴风险——没有哪条路是完美的，就看自己更看重什么——我选择自由！

反混沌

在 Handie 上除了继续优化并完善配置化与跨 JS-based 技术栈方面的能力，还会去探索挖掘前端开发及团队协作工作流相关的功能特性。

将「反混沌」相关的事物进一步整合，并与「FES」进行联动，争取打造成一个高质量的知识型社区！

个人服务

最主要的就是开始塑造上文所说的个人知识库产品 KnoSys，其次的话——

基于物理家庭（房子）的多中心个人「云」服务器——

使其作为个人与家庭的唯一可信（SSOT）数据中心，让手机、MacBook、PC 等基本只作为单纯消费数据的终端存在，不在它们上面存储数据，省去数据冗余和同步问题，减少各终端存储空间的浪费。

个人「云」服务器上基于 Docker 搭建各种服务，用于个人知识库体系、家庭媒体中心、家庭事务管理系统、智能家居控制中心等。

不行……脑洞太多，激动得我快尿了……

欧雷的想法

点子有很多，至于会去做几个以及能做到什么程度，随缘，绝不进行自我剥削！

总结

就用一句话来总结这一年以及对未来的态度吧——

认清现实不是为了妥协，而是更好地追梦。オレは負けない。
